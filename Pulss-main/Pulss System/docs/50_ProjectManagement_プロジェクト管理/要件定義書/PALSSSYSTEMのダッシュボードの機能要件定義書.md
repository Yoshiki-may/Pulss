0. 前提・スコープ
0-1. 対象範囲

対象：

営業〜CS〜ディレクション〜クリエイターが使う「社内向けダッシュボード」

クライアント向けの「閲覧専用レポート画面（＋PDF/CSV 出力）」

対象業務：

リード/案件管理（営業）

商談内容・ヒアリング結果の管理

提案書・契約前のフォロー管理

契約後のキックオフ・引き継ぎ

制作〜投稿までのタスク管理（誰がボールを持っているか）

月次効果測定レポート（インサイト・売上などの数値＋コメント）

非対象（本ドキュメントでは詳細設計しない）：

AI による文章生成ロジック

AI による要約・企画案生成ロジック

モデル選定や API 料金などの AI 基盤部分

第8章 通知・アラート機能

最低限のアラートは初期から欲しい、というニーズ。

8-2. 通知トリガー例（初期）

第9章 外部サービス連携（SNS・チャット・NAS）
9-1. SNS連携（インサイト取得）

初期は「手動数値入力」を許容しつつ、将来的に以下を想定：

Instagram / TikTok / YouTube / 公式LINE などのインサイト API 連携

クライアント別に

アカウントID / トークンを登録

効果測定ダッシュボードで自動取得

※利用者希望：「既存のインサイト可視化ツールを参考に、UI/UXはそれ以上に綺麗に」

9-2. チャット連携

現状：

Slack、Teams、Chatwork、個人LINEなどクライアントごとにバラバラ

短期（フェーズ1）

クライアントごとに「チャット種別＋リンク」を登録できるメモ欄

ダッシュボードからワンクリックで該当チャットへ飛ぶ程度

中期（フェーズ2）

公式LINE / LINE@ に集約できたアカウントは、メッセージ履歴をダッシュボード側で閲覧

「未返信◯日」のスレッドを検知してエスカレーション

9-3. NAS／社内ストレージ連携

目的：

重い動画ファイルのやり取りをNASで行い、その「受け渡し完了状態」だけをダッシュボードで管理する。

要件

クライアント毎に「NAS上のフォルダパス」を1つ紐づけ

ダッシュボード側では：

「素材アップ済み」「編集済み」「納品済み」といったステータスをボタン操作で管理

実際のファイル操作はOSレベルで行う

優先度

P0: チャットURLメモ欄、NASフォルダパス管理

P1: 公式LINE 連携

P2: SNS API連携
