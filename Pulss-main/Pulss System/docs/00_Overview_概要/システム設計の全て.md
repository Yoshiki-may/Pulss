データとAIを中心にした “売れるコアシステム” と運用体制を作る

A. 共通：アカウント・案件基盤

A-1. アカウント構造

[MVP] 「クライアント企業」＞「アカウント（ブランド単位）」＞「案件（キャンペーン単位）」の階層

[MVP] 各案件に以下の基本情報を保持

目的（採用／集客／ブランディング 等）

担当営業／担当ディレクター／担当編集者

[P1] 「採用と集客」など複数目的を持つ案件にも対応できるタグ構造

A-2. 権限・ロール管理

[MVP] ロール：営業・ディレクション・クリエイティブ・CS・管理者・クライアント

[MVP] ロールごとに閲覧権限を制御（例：営業のランキングは営業のみ閲覧可など）

[P1] 重要クライアントのみ「限定公開情報（原価など）」領域を用意

4. 主な業務フロー（システム説明用）
フロー1：新規リード〜契約〜要件定義

営業がリードを CRM に登録

クライアントが PALSSチャット（AIヒアリング）で事前回答

営業がその結果を元に商談 → 補足ヒアリング

商談内容＋AIヒアリング結果から、
AIが要求定義レポート→要件定義書を作成

ディレクション（山崎さん側）が要件定義をチェックし、必要に応じて修正

フロー2：制作ワークフロー

要件定義書を元に、企画・台本案をAI＋人間で作成

撮影計画を立て、カメラマン・編集者をアサイン

撮影 → 素材をNASに格納

編集者がクリエイティブダッシュボードから指示・リファレンスを確認し編集

社内チェック → クライアントダッシュボードで承認

承認後、投稿スケジュールに従いSNSへ投稿

フロー3：月次レポート・効果測定

システムがプラットフォームからインサイトデータ取得

AIが1ヶ月の結果をまとめ、月次レポートの素案を生成

ディレクション／CSが必要な追記を行い、クライアントへ共有

クライアントが事業側指標（売上・採用数など）を入力

優良事例は自動的に「ケーススタディ候補」としてナレッジベースへ

フロー4：教育・コンプライアンス

営業／ディレクションの現場会話を録音 → 文字起こし

幹部・教育担当が問題点・良い点をコメント

フィードバック履歴を人ごとに蓄積

定期的なロープレや教育に活用

重大クレーム時には該当ログを参照して再発防止策へ
