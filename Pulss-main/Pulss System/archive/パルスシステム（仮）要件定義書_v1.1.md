# 0. メモ：AI／n8n 活用ポイントまとめ

## 0-1. プリセールス〜受注

- **AIでやること**
  - パルスチャット回答からの要件定義サマリ生成
  - ブランド要約／SNS戦略方針／初期コンテンツ案（例：10本）
  - 商談議事録の要約・論点整理・ネクストアクション抽出
  - 提案書ドラフト（章立て・ストーリー）自動生成

- **n8nでやること**
  - パルスチャット完了 → Webhook で受信 → LLM API 呼び出し → DB 保存 → 営業・ディレクターに通知
  - 議事録テキスト or 音声文字起こしをトリガーに要約フロー実行
  - 提案書ドラフトを Google Docs / Notion / PowerPoint テンプレに自動書き込み
  - CRM やスプレッドシートへのリード自動登録

---

## 0-2. 受注後〜制作・運用

- **AIでやること**
  - 案件属性からの担当アサイン候補（ディレクター／編集者）のスコアリング（将来）
  - 撮影指示書・ショットリスト・編集トーンのドラフト
  - コンテンツカードの企画タイトル・フック案・テロップ文案ドラフト

- **n8nでやること**
  - 「案件ステータス＝受注」をトリガーに案件レコード自動作成
  - タスク管理ツール（Notion / ClickUp / Asana 等）へのタスク自動発行
  - S3 等への素材アップロードをトリガーに AI編集サーバへジョブ投げ
  - ステータス更新（例：AI下処理中 → 人間仕上げ中）

---

## 0-3. 運用・レポーティング

- **AIでやること**
  - KPIトレンド分析（今月のハイライト／課題／次の一手）
  - 月次レポートのテキストドラフト（サマリ・インサイト・提案）

- **n8nでやること**
  - 各SNS API からのインサイト定期取得バッチ
  - 投稿数／インプレッション／エンゲージメントの集計
  - KPIの閾値判定→アラート発火（健康スコア・解約リスク）
  - レポートPDF生成→メール／チャットでの送付

---

## 0-4. 議事録・提案資料自動生成

- **AIでやること**
  - 商談音声の要約・論点整理・決定事項・ネクストアクション抽出
  - 追加施策提案・提案書章立て自動生成

- **n8nでやること**
  - 録音完了 or 文字起こし完了をトリガーに AI 要約フロー実行
  - 議事録・提案ドラフトを Notion / Google Docs へ保存
  - 営業・ディレクターへ「ドラフトできたよ」通知

---

## 0-5. AI編集支援（ヒロスタイルのシステム化）

- **AIでやること**
  - ヒロ編集動画のスタイル分析（色味・カットテンポ・音ハメ傾向）
  - 業種・ブランドトーン別「ヒロ風プリセット」生成
  - テロップ文案・構成ドラフト生成

- **n8nでやること**
  - 素材アップロード → スタイル推定 → プリセット適用 → ラフ版出力のジョブ連携
  - ジョブ失敗・遅延時のアラート
  - 修正依頼コメント → AI要約 → 編集者タスク登録

---

## 0-6. 通知・エスカレーション

- **AIでやること**
  - クライアントからのテキスト（クレーム・長文要望）の感情分析・緊急度スコアリング
  - 長文メッセージの要約（担当者がすぐ状況把握できるように）

- **n8nでやること**
  - 「24時間未返信」「48時間未返信」「7日投稿なし」等のルール監視
  - Slack / LINE / メールなどマルチチャネル通知
  - エスカレーション先の切り替え（ディレクター → 営業 → 役員）

---

## 0-7. AI基盤・権限・マルチテナント

- **AIでやること**
  - 業界別エキスパートAI（電気会社／引っ越し／テーマパークなど）
  - 外部LLM → 自社LLM への切り替え（推論先の統一インターフェース）

- **n8nでやること**
  - LLMエンドポイントを環境変数で切り替え（外部API or 自社GPU）
  - 全てのワークフローにテナントIDを通し、データ分離のルールを担保

# 1. パルスシステム（仮）要件定義書 v1.0  
映像制作・SNS運用事業のスケールとバイアウト価値向上のための AI＋ワークフロー基幹システム

---

## 1. 背景・目的

### 1-1. 現状課題（As-Is）

- 営業は高い獲得力（1人で月15件以上）があるが、制作・ディレクションがボトルネックとなり、意図的に営業を絞っている。
- トップクリエイター「ヒロ」への属人化が大きく、「ヒロが止まると会社が止まる」リスクが高い。
- 若手ディレクターが「業務連絡係」化しており、提案・ブランド伴走が弱い。
- 経営者（レン君）にクライアント連絡が集中し、現場から離れられない。
- 将来 500 アカウントを目指す中で、現行フローだと 200 アカウント程度が限界。

### 1-2. 中長期目標

- 3期目までに：
  - 500アカウント
  - 売上 約17億
  - 営業利益率 60％
- 属人性を下げ、売却マルチプルを **3倍 → 5〜7倍** に引き上げ、「2年10ヶ月でのバイアウト」を狙う。

### 1-3. パルスシステム（仮）の目的（To-Be）

- **営業を止めずに伸ばせる状態を実現**
  - ディレクターの「聞く力」をAI化し、商談前ヒアリング・要件定義を自動で高水準に揃える。
- **少人数ディレクションでの高LTV運用**
  - 100〜500アカウントを10名程度のディレクション/CSで運用可能にする。
- **ヒロのクリエイティブの“システム化”**
  - AI編集支援により編集工数の60%を自動化し、残り40%を人間の感性に集中。
- **売却しやすい会社構造の構築**
  - 業務フロー・ナレッジ・データを1つのプラットフォームに集約し、「人ではなくシステムを買う」状態を作る。

---

## 2. コンセプト・全体アーキテクチャ

### 2-1. サービスコンセプト

- キーワード：**「AIディレクター × クリエイティブOS」**
- SNS運用・映像制作の
  - 営業
  - ディレクション
  - 撮影
  - 編集
  - レポーティング
  を一気通貫で支える **社内専用 AI＋業務基盤システム**。

- 将来的には：
  - 自社用の基幹システム
  - 他社のSNS代行・制作会社にも展開可能なSaaSプロダクト
  として再利用できる設計を想定。

### 2-2. アーキテクチャ構成（概念）

1. **アプリケーション層**
   - パルスチャット（クライアント向けAIヒアリングUI）
   - 各種ダッシュボード（営業／ディレクター／クリエイター／クライアント／CS）
   - コンテンツカード管理、投稿管理UI

2. **AIサービス層**
   - LLM（外部API or 自社LLM）
   - 議事録要約・提案書生成サービス
   - AI編集スタイル分析サービス
   - KPI分析・解約リスクスコアリングサービス

3. **ワークフロー・連携層（n8n）**
   - フォーム送信やステータス更新をトリガーとした処理自動化
   - SNS API連携、CRM連携、カレンダー連携
   - 通知・エスカレーション、バッチ処理

4. **データ層**
   - 案件・アカウント・コンテンツ・KPI・ログを一元管理するDB
   - オブジェクトストレージ（動画・素材・サムネイル）

### 2-3. 想定ユーザー・ロール

- 経営者 / 役員  
  → 全体KPI／LTV／解約リスクの俯瞰、リソース配分、意思決定
- 営業（クロージャー）  
  → 商談前ヒアリング確認、提案書の自動生成、契約・金額調整
- ディレクター  
  → 要件定義の補正、コンテンツ企画、クライアント窓口
- カメラマン  
  → 撮影スケジュール確認、指示書（絵コンテ・ショットリスト）確認
- 編集者  
  → 編集指示、AIプリセットの適用、修正対応
- CS（カスタマーサクセス）  
  → アカウントの健康状態監視、アップセル・クロスセル提案
- クライアント担当者  
  → 依頼／要望送信、コンテンツ確認・承認、成果レポート閲覧
- システム管理者  
  → 権限管理、マスタ管理、AI学習用データの管理

---

## 3. 業務フロー（To-Be）

### 3-1. プリセールス〜受注

1. リード獲得（テレアポ／紹介）
2. 営業が「パルスチャット」URLを送付
3. クライアントがチャット形式で回答
4. AIが要件定義サマリ＋初期コンテンツ案を自動生成
5. 商談（オンライン／対面）
   - 録音→文字起こし→要約
   - n8nで議事録・提案書ドラフト自動生成
6. 提案→受注
   - 受注確定と同時に案件レコード自動作成（n8n）

### 3-2. 受注後〜制作・運用

1. 担当割り当て
   - ディレクションダッシュボード上で担当者を決定
   - 将来的にはAIによるアサイン候補提示
2. 撮影準備
   - AIが撮影指示書・ショットリスト・編集トーンをドラフト生成
3. 撮影→素材アップロード
4. AI編集×人間仕上げ
   - AIが 60% ラフ版生成
   - 編集者が 40% 仕上げ
5. クライアント承認
6. 投稿カレンダー管理・自動／手動投稿

### 3-3. 運用・レポーティング

1. SNSインサイト自動取得（n8nバッチ）
2. KPIダッシュボードでの見える化
3. 月次レポート自動ドラフト＋ディレクターコメント
4. 健康スコア・解約リスクアラート（n8n + AI）発火

---

## 4. 機能要件

### 4-1. パルスチャット（AIヒアリング）

**目的**  
ディレクターの「聞く力」をテンプレ化し、誰が担当しても必要十分なヒアリングができる状態を作る。

**機能要件**

- チャットUIでの会話型ヒアリング（PC／スマホブラウザ対応）
- 途中保存・再開機能
- 業種ごとの質問テンプレート
  - 焼肉／ホテル／美容／テーマパーク／引っ越し 等
- 取得項目例：
  - 業種／エリア／店舗数
  - 単価帯・ターゲット属性
  - ブランドトーン（カジュアル／ラグジュアリー 等）
  - 競合・ベンチマーク
  - 押したい商品／サービス
  - NG・避けたい表現
  - 参考動画・アカウントのURL添付
- AIサマリー生成：
  - ブランド要約
  - SNS戦略方針
  - 初期コンテンツ案（例：10本）

**n8n連携**

- パルスチャット完了 → n8n Webhook 呼び出し
- n8n側処理：
  - LLM API呼び出しで要約生成
  - DBへ保存
  - 営業・ディレクターへ通知
  - 必要に応じて CRM へのリード自動登録

---

### 4-2. ダッシュボード（4視点）

1. **営業／ディレクション用ダッシュボード**
   - 案件一覧（フェーズ：リード／商談中／提案中／受注／解約）
   - 本日〜7日間の対応タスク
   - 重要アラート
     - 未返信問い合わせ
     - 未承認コンテンツ
     - 投稿スケジュール未登録 等

2. **クリエイティブ用ダッシュボード（カメラマン・編集者）**
   - 自分の担当案件・タスク一覧
   - 撮影日カレンダー
   - 編集ステータス（未着手／AI下処理中／人間仕上げ中／レビュー中／完了）

3. **クライアント用ダッシュボード**
   - 投稿カレンダー（過去＆未来）
   - 成果メトリクス（閲覧数／いいね／保存／予約数 等）
   - 制作中コンテンツのプレビュー＆承認
   - 相談・要望投稿フォーム

4. **CS／管理者用ダッシュボード**
   - 全アカウントの健康スコア表示（例：緑・黄・赤）
   - 解約リスク予測（投稿頻度／指標悪化／返信遅延など）
   - 人員リソース負荷（営業／ディレクター／編集者ごとの案件数）

**n8n連携**

- ダッシュボードは n8n による集計結果（KPI・アラート）を反映して表示する。

---

### 4-3. ワークフロー管理・通知・エスカレーション

**機能要件**

- ステータス管理
  - 案件／コンテンツ／タスクにステータスを付与  
    （要件定義中／撮影準備／撮影済／編集中／クライアント確認中／投稿済 等）
- 通知機能
  - 画面内通知（バッジ・サウンド）
  - メール／LINE／Slack 連携
- エスカレーションルール例：
  - クライアントからのメッセージ未返信 24時間 → ディレクター通知
  - 48時間未返信 → 営業／CSへエスカレーション
  - 撮影日前日までに指示書未確定 → アラート
- 対応履歴の記録
  - 誰が／いつ／どのアラートにどう対応したかログ保存

**n8n連携**

- ルールベースの監視ロジックは n8n フローとして実装
- 各種通知チャネルへの送信も n8n で実行

---

### 4-4. コンテンツ制作・承認・投稿管理

**コンテンツカード**

- 1投稿（リール／ショート等）を「コンテンツカード」として管理
- 要素：
  - 企画タイトル
  - 目的・KPI
  - 撮影日／撮影場所
  - 使用ショット一覧
  - BGM／SE 情報

**撮影指示書／ショットリスト**

- パルスチャット＆過去コンテンツを元にAIがドラフト生成
- ディレクターが人間の目でチェック・修正

**編集フロー**

- 素材アップロード（S3等のオブジェクトストレージ）
- AIによる「60%仕上げ」（カット・カラーのたたき台）
- 編集者による 40% のクリエイティブ仕上げ
- バージョン管理（v1, v2, ...）

**クライアント承認**

- クライアントダッシュボードでプレビュー
- 「承認／修正依頼」の二択＋コメント欄
- 修正依頼コメントはタスクとして編集者ダッシュボードに反映

**投稿管理**

- 投稿予定カレンダー表示
- API経由での自動投稿（対応プラットフォームから順次）
- 手動投稿の場合は「実績投稿日時」を記録

**n8n連携**

- 素材アップロード（S3イベント／Webhook）をトリガーにAI編集ジョブ起動
- 投稿時刻になったら n8n が各SNS APIを叩いて自動投稿
- 投稿結果／エラーをログ化・通知

---

### 4-5. KPI・効果測定・レポート

**初期要件**

- SNSインサイト取得
  - 投稿数
  - インプレッション／リーチ
  - エンゲージメント（いいね・コメント・シェア・保存）
- ダッシュボードでの可視化
  - 月次・週次・日次フィルタ
  - 投稿別ランキング表示
- レポート自動生成
  - AIが月次レポートのドラフト文章を生成
  - ディレクターがコメント追加し、PDF生成

**将来要件**

- 売上連動型のレポート（予約システム・POS連携）
- 「解約リスクスコア」の自動算出

**n8n連携**

- スケジュール実行（毎日／毎月）
  - SNS API → DB → 集計
  - LLMでレポート文章生成 → PDF or スライド出力
  - クライアントへメール送付 or ダッシュボード公開

---

### 4-6. 議事録・提案資料自動生成

**機能要件**

- 商談／定例ミーティングの音声録音・文字起こし（外部API利用）
- 要約・ネクストアクション抽出
  - 「概要」「重要ポイント」「決定事項」「次回アクション」
- 議事録テンプレート（レン君の会社向けフォーマット）
- 提案書・社内共有資料の自動生成
  - 議事録＋パルスチャット情報から初回提案資料／追加施策提案を生成

**n8n連携**

- 録音完了→n8nトリガー
- 文字起こしAPI→LLM要約→議事録・提案ドラフト作成
- Google Drive / Notion / 自社画面への保存
- 営業／ディレクターへの通知

---

### 4-7. AI編集支援（ヒロスタイルのシステム化）

**機能要件**

- スタイル分析
  - ヒロ編集動画群から、カラーグレーディング／カットテンポ／音ハメ傾向を数値化
- AIプリセット生成
  - 業種・ブランドトーン別「ヒロ風プリセット」を生成
  - 編集ソフト（Premiere / DaVinci 等）の LUT・テンプレートとして出力
- 自動プリセット適用
  - 素材動画にプリセットを適用し「60%完成」ラフ版を自動作成
- 編集者UI
  - タイムライン表示
  - カット・色味・テロップなど人間側で微調整

**n8n連携**

- 素材アップロード → スタイル推定 → プリセット適用 → ラフ版出力のジョブ管理
- ジョブ失敗時のアラート発火
- 修正依頼コメント → AI要約 → 編集者タスク登録

---

### 4-8. 権限管理・監査

**機能要件**

- ロールベースアクセス制御（RBAC）
  - 営業／ディレクター／編集者／クライアント／管理者で閲覧範囲・編集権限を制御
- ログ・監査
  - 重要操作（契約条件変更／クレーム対応／承認操作など）の履歴を保存
- NDA／クライアント個別モデル用のデータ分離
  - 特定クライアントのノウハウを学習したAIは他社へ出力しないよう制御

---

### 4-9. AI基盤・スパコン活用

**初期フェーズ**

- 外部LLM（例：OpenAI API 等）の活用を前提
- APIコストを監視しながら PoC／β版を構築

**スパコン導入フェーズ**

- 自社GPUサーバ上にオープンソースLLMを構築
- 自前インフラ上での推論を可能にし、API従量課金を圧縮／ゼロ化

**業界別エキスパートAI**

- 電気会社、引っ越し会社、テーマパーク業界など
- それぞれのノウハウ・データを学習した「業界別エキスパートAI」を構築
- クライアントとのNDAに基づきデータを取り扱う

**n8n連携**

- LLM推論先（外部API or 自社GPU）を環境変数で切り替え
- 全ワークフローでテナントID・クライアントIDを通し、データ分離を担保

---

## 5. 非機能要件（NFR）

- **パフォーマンス**
  - 同時ログインユーザー：社内 50名＋クライアント 500名 を想定
  - 通常操作で 2秒以内の応答を目標
- **可用性**
  - 目標稼働率：99.5%以上
- **セキュリティ**
  - 通信の暗号化（HTTPS/TLS）
  - アカウント毎のデータ分離（マルチテナント設計）
  - ログイン時の多要素認証（将来要件）
- **スケーラビリティ**
  - 500アカウントまでスケールした状態でもダッシュボードのレスポンスが許容範囲に収まること
- **保守性**
  - マイクロサービス／モジュール構成
  - n8nを「外部連携・オートメーション層」として明確に分離
- **拡張性**
  - 業界依存ロジックはテンプレート・設定として外出し
  - 将来的に他事業（不動産FC、別業界のSES事業など）にも使い回しができるようにする

## 7. 組織・運用インパクト

- **ディレクションの再定義**
  - 「営業の秘書」「CS＋提案の窓口」にシフト
  - 日程調整・業務連絡の多くはAI＋システムに任せる
- **営業の最大化**
  - 営業人数を増やしてもディレクションがボトルネックにならない構造へ
  - レン君は営業・採用から離れ、CS／グロース専任へ
- **編集・クリエイティブ**
  - ヒロが“高級食材”として、重要クライアント・新しい表現の開発に集中できる状態を作る
- **バイアウト価値**
  - 「人に依存しない運用モデル」と「AI＋データを内包したシステム」をセットで売れる状態を作る
